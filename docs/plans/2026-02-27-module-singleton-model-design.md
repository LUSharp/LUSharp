# Module/Singleton Model Redesign

**Date:** 2026-02-27
**Scope:** Plugin (`plugin/src/`) — Lowerer, Emitter, ScriptManager, EditorTextUtils, IntelliSense

## Problem

The current transpiler model uses Script/LocalScript/ModuleScript distinctions based on base class, has a `GameEntry()` entry point convention, no inter-script imports, no service hoisting, and several bugs (auto-indent, namespace diagnostics, hover resolution).

## Design

### Part 1: Script Model Overhaul

Every compiled script becomes a ModuleScript that returns its class table. No more Script/LocalScript distinction in transpiler output.

**Changes:**
- `GameEntry` renamed to `Main()` everywhere (templates, entry detection, Emitter)
- Emitter always appends `return ClassName` at the end
- Lowerer drops `SCRIPT_BASES` / `determineScriptType` — `scriptType` is always `"ModuleScript"`
- ScriptManager templates updated: `Main()` replaces `GameEntry()`

**Compiled output:**
```lua
local MyClass = {}

function MyClass.new()
    local self = {}
    return self
end

function MyClass:Main()
    -- entry point
end

return MyClass
```

### Part 2: Lazy Loader

A single runtime Script that manages the module lifecycle:

1. **Discover:** CollectionService finds all `LUSharp`-tagged ModuleScripts
2. **Require:** Requires each, getting back the class table
3. **Instantiate:** Calls `.new()` on each for a singleton, stores in registry
4. **Run:** Calls `:Main()` on any singleton that has it

```lua
local CollectionService = game:GetService("CollectionService")

local _registry = {}
local _modules = {}

for _, instance in CollectionService:GetTagged("LUSharp") do
    if instance:IsA("ModuleScript") then
        local classTable = require(instance)
        _modules[instance.Name] = classTable
    end
end

for name, classTable in _modules do
    if type(classTable.new) == "function" then
        _registry[name] = classTable.new()
    else
        _registry[name] = classTable
    end
end

for name, instance in _registry do
    if type(instance.Main) == "function" then
        instance:Main()
    end
end
```

This is a static Lua file shipped with the plugin/runtime, not generated by the transpiler. Lives in a `runtime/` folder mapped via Rojo.

Circular deps at the loader level are not an issue: all modules are required (Phase 2), then instantiated (Phase 3), then run (Phase 4). By the time `Main()` executes, everything is loaded.

### Part 3: Smart Require Imports + Circular Dependency Guard

#### Require Generation

When a script has `using Game.Server;` and references a class from that namespace:

1. Parser collects `ast.usings` (namespace imports)
2. Lowerer scans identifiers used in method bodies
3. Cross-references against classes known in imported namespaces
4. Emits `require()` only for modules actually referenced
5. Require paths use service locals (from Part 5)

#### Circular Dependency Guard (C++-style include guard)

Every emitted module gets a guard at the top, preventing re-entrant require chains:

```lua
-- Circular dependency guard
if _G.__LUSharp_Loading == nil then _G.__LUSharp_Loading = {} end
if _G.__LUSharp_Loading["MyClass"] then return _G.__LUSharp_Loading["MyClass"] end
_G.__LUSharp_Loading["MyClass"] = {}

-- Services
local Players = game:GetService("Players")

-- Requires
local OtherClass = require(...)

-- Class definition
local MyClass = {}
function MyClass.new() ... end
function MyClass:Main() ... end

-- Register and return
_G.__LUSharp_Loading["MyClass"] = MyClass
return MyClass
```

**How it works:**
- A requires B -> B starts loading -> B requires A -> A is already in `_G.__LUSharp_Loading` (placeholder `{}`) -> returns immediately
- When A finishes loading, it overwrites the placeholder with the real table
- Since Lua tables are references, B's local pointing to the placeholder now points to the real table
- Zero runtime overhead after first load

**Circular dependency warnings:** At the plugin level (IntelliSense), detect mutual `using` references between scripts and emit a warning diagnostic so the user is aware.

### Part 4: Bug Fixes

#### Auto-Indent (EditorTextUtils.lua)

**Bug:** After pressing Enter inside a namespace or base class block on an empty line, cursor goes to column 1 with no indentation. Works correctly for `foreach` because the previous line ends with `{`.

**Root cause:** `computeAutoIndentInsertion` (line 399-418) only adds indent when `trimmed:sub(-1) == "{"`. On an empty line, it falls to the `trimmed == ""` branch which does brace-depth counting via tokenization, but the depth computation is incorrect when namespace/class nesting is involved.

**Fix:** Always compute brace depth from the full token stream up to the cursor position, regardless of whether the previous line is empty or not. Remove the special-case branching. If the previous non-empty line ended with `{`, the depth scanner already accounts for it.

```lua
local desiredIndent
local tokens = Lexer.tokenize(newText:sub(1, insertedNewlinePos - 1))
local depth = 0
for _, token in tokens do
    if token.type == "punctuation" then
        if token.value == "{" then
            depth += 1
        elseif token.value == "}" then
            depth = math.max(0, depth - 1)
        end
    end
end
desiredIndent = string.rep(tabText, depth)
```

#### Namespace Diagnostic (IntelliSense.lua)

**Bug:** `using System;` shows error until a second sub-namespace is imported.

**Root cause:** `getUsingNamespaceNames()` builds known namespaces from `DOTNET_NAMESPACE_MEMBERS` keys and `TypeDatabase.types` namespaces via `addNamespaceWithParents`. The root `System` namespace may not be getting registered because `addNamespaceWithParents` only adds parent segments, and the direct key `System` in `DOTNET_NAMESPACE_MEMBERS` might not be added to the `seen` set properly.

**Fix:** Ensure `getUsingNamespaceNames` explicitly adds each key of `DOTNET_NAMESPACE_MEMBERS` as a known namespace, not just their parents.

#### Hover Resolution (IntelliSense.lua)

**Bug:** Clicking on `Server` in `Game.Server` only shows `Game`.

**Root cause:** Hover resolver splits on `.` and resolves only the token under the cursor instead of the full dot-path.

**Fix:** When resolving hover info, walk backwards and forwards from cursor to collect the full dot-delimited path (e.g., `Game.Server`), then resolve the complete path.

### Part 5: Service Optimization Pass

A new pass in the Lowerer that runs after all expressions are lowered:

1. **Collect** all `game:GetService("X")` and `game.GetService<X>()` calls in the module IR
2. **Deduplicate** by service name
3. **Emit** `local ServiceName = game:GetService("ServiceName")` at the very top of the module
4. **Replace** every `game:GetService("X")` in the IR body with the identifier `ServiceName`

**Emitted output ordering:**
```
1. Circular dependency guard
2. Service locals (hoisted)
3. Require locals (reference service locals for paths)
4. Class definition
5. Register + return
```

**Implementation in Lowerer:**
- Add `collectServiceCalls(module)` that walks the IR tree
- Returns a deduplicated list of `{ name = "Players", original = <IR node> }`
- Add `hoistServices(module, services)` that prepends `local_decl` statements and replaces original nodes with identifiers
- Run after `lowerModule` returns, before emitting
